# Скрипты для управления IoT Platform

В этой папке находятся скрипты для управления IoT Platform.

## Доступные скрипты

### `start.sh` - Запуск проекта
Запускает все сервисы IoT Platform:
- Проверяет наличие Docker и Docker Compose
- Проверяет наличие файла `.env`
- Останавливает существующие контейнеры
- Собирает и запускает все сервисы
- Ожидает готовности PostgreSQL
- Выводит информацию о доступных сервисах

**Использование:**
```bash
./start.sh
```

### `stop.sh` - Остановка проекта
Останавливает все сервисы IoT Platform с сохранением данных:
- Останавливает все контейнеры
- Проверяет статус после остановки
- Сохраняет данные в volumes

**Использование:**
```bash
./stop.sh
```

### `clean.sh` - Остановка и очистка
Останавливает все сервисы и полностью удаляет данные:
- Запрашивает подтверждение перед удалением данных
- Останавливает все контейнеры
- Удаляет volumes с данными (postgres_data, blackbox_models)
- Очищает неиспользуемые Docker ресурсы
- Полностью удаляет все данные из базы данных

**Использование:**
```bash
./clean.sh
```

## Примеры использования

### Запуск проекта
```bash
cd project_v2/iot_platform/scripts
./start.sh
```

### Остановка проекта (с сохранением данных)
```bash
./stop.sh
```

### Полная очистка данных
```bash
./clean.sh
```

## Доступные сервисы после запуска

- **Backend API:** http://localhost:8000
- **API Документация:** http://localhost:8000/docs
- **BlackBox ML:** http://localhost:8001
- **ML API Документация:** http://localhost:8001/docs
- **Frontend:** http://localhost:3000
- **Симулятор:** http://localhost:5001
- **PostgreSQL:** localhost:5432

## Полезные команды

```bash
# Просмотр логов всех сервисов
docker compose -f ../docker-compose.yml logs -f

# Просмотр логов конкретного сервиса
docker compose -f ../docker-compose.yml logs -f backend

# Проверка статуса сервисов
docker compose -f ../docker-compose.yml ps

# Перезапуск конкретного сервиса
docker compose -f ../docker-compose.yml restart backend
```

## Примечания

- Все скрипты должны быть запущены из папки `scripts`
- Скрипты автоматически переходят в корневую директорию проекта для выполнения команд Docker Compose
- Убедитесь, что файл `.env` существует в корневой директории проекта перед запуском
